THIẾT KẾ GIAO DIỆN HỆ THỐNG QUẢN LÝ NHÂN SỰ (EMS)
Tài liệu Đặc tả Yêu cầu UI/UX và Kiến trúc Frontend dành cho AI Code Generator

1. TỔNG QUAN HỆ THỐNG (SYSTEM OVERVIEW)
Framework Core: Next.js (App Router), React.

Ngôn ngữ: TypeScript (Bắt buộc tuân thủ các Interface được định nghĩa bên dưới).

Styling: Tailwind CSS kết hợp thư viện UI shadcn/ui (sử dụng Table, Card, Dialog/Sheet, Form, Badge, Toast).

Quản lý trạng thái (State): Zustand hoặc React Context để lưu trữ session người dùng toàn cục.

Cơ chế xác thực: JWT (JSON Web Token) được lưu tại HttpOnly Cookie. Mọi request gọi API (thông qua Axios Interceptor) đều phải đính kèm header Authorization: Bearer <token>.

2. ĐỊNH NGHĨA KIỂU DỮ LIỆU (TYPESCRIPT INTERFACES)
AI cần tuân thủ nghiêm ngặt các interface này để đảm bảo đồng bộ với cấu trúc cơ sở dữ liệu Oracle:

// Payload giải mã từ JWT
export interface UserSession {
  userId: string;
  username: string;
  role: 'STAFF' | 'MANAGER' | 'HR_STAFF' | 'HR_MANAGER' | 'ACCOUNTANT';
  maPB: 'BOD' | 'HR' | 'IT' | 'ACC' | 'MKT';
}

// Bảng NHAN_VIEN
export interface Employee {
  maNV: string;
  hoTen: string | '***'; // Nhận '***' nếu bị Redaction
  ngaySinh: string | '***'; 
  sdt: string | '***';
  luong?: number; // Có thể null/undefined nếu Role là STAFF
  maSoThue: string;
  maPB: string;
}

// Bảng DU_AN
export interface Project {
  maDA: string;
  tenDA: string;
  nganSach: number;
  ngayBatDau: string;
  trangThai: string;
  olsLabel: 'PUB' | 'CONF' | 'SEC'; // Nhãn bảo mật Oracle Label Security
}

3. PHÂN LỚP QUYỀN HẠN & ĐIỀU KIỆN RENDER GIAO DIỆN
AI cần áp dụng các điều kiện này (Conditional Rendering) vào giao diện tương ứng với thuộc tính role của UserSession:

Role (Vai trò),View Cột Lương,View Dữ liệu Định danh,Thao tác (Form Edit),View Dự Án (OLS),Dashboard FGA
STAFF,ẨN (Không render),Hiển thị bình thường,Không có nút Edit,Nhãn PUB,Chặn (403)
MANAGER,HIỂN THỊ,Hiển thị bình thường,Không có nút Edit,"Nhãn PUB, CONF",Chặn (403)
HR_STAFF,HIỂN THỊ,Hiển thị bình thường,"Được Edit, nhưng Input Lương bị khóa (Disabled)","Nhãn PUB, CONF",Chặn (403)
ACCOUNTANT,HIỂN THỊ,Che mờ (Masking: ***),Không có nút Edit,"Nhãn PUB, CONF",Chặn (403)
HR_MANAGER,HIỂN THỊ,Hiển thị bình thường,Được Edit mọi field,"Full (PUB, CONF, SEC)",Cho phép


4. ĐẶC TẢ CHI TIẾT CÁC MÀN HÌNH (SCREENS & VIEWS)
4.1. Màn hình Đăng nhập (Login Screen)
Route: /login

UI Components: Form đăng nhập canh giữa màn hình. Gồm Input (Username), Input type="password" (Password).

Logic: Nút "Login" có trạng thái Loading spinner. Gọi API POST /api/auth/login. Thành công -> lưu UserSession -> Redirect /employees.

4.2. Màn hình Danh sách Nhân sự (Employee Directory)
Route: /employees

UI Components: DataTable với thanh tìm kiếm và phân trang.

AI UI Generation Rules (Quy tắc sinh giao diện):

Cột Mức Lương: Wrap header và cell bằng điều kiện {session.role !== 'STAFF' && <TableColumn>...}.

Dữ liệu Redaction (Kế toán): Tại các ô Họ Tên, SĐT, Ngày Sinh, nếu giá trị trả về là ***, render component <Badge variant="secondary">Dữ liệu ẩn</Badge> để minh họa tính năng Data Masking của CSDL.

Cột Hành động (Edit): Cột này chứa nút icon ✏️, chỉ hiển thị nếu ['HR_STAFF', 'HR_MANAGER'].includes(session.role).

4.3. Biểu mẫu Cập nhật Nhân sự (Employee Edit Form)
Route: Sheet (Drawer trượt từ phải) hoặc Dialog modal tại trang /employees.

UI Components: Form nhập liệu.

AI UI Generation Rules:

Trường "Mức lương": Thiết lập <Input type="number" disabled={session.role === 'HR_STAFF'} />. Hiển thị thêm dòng text nhỏ: "Nhân viên HR không có quyền sửa lương" nếu disabled.

Error Handling: Khi gọi API PUT /api/employees/:id, nếu Backend trả về lỗi HTTP 403 (Oracle Trigger chặn), kích hoạt <Toast variant="destructive"> hiển thị thông báo: "Từ chối truy cập: Vi phạm chính sách bảo mật cơ sở dữ liệu."

4.4. Màn hình Lịch sử Biến động Lương (Salary History Timeline)
Route: /employees/[id]/salary-history

UI Components: Component Vertical Timeline (Trục thời gian).

Data UI: Mỗi node lịch sử render: [Icon Mũi tên tăng/giảm] + Ngày đổi + Người thực hiện (ID) + <span className="text-green-600">Lương cũ ➔ Lương mới</span>.

4.5. Màn hình Quản lý Dự án (Projects View - OLS Security)
Route: /projects

UI Components: Render danh sách dự án dạng lưới thẻ (Card Grid).

AI UI Generation Rules (Trọng tâm hiển thị OLS):

Dựa vào project.olsLabel, render các <Badge> ở góc trên thẻ dự án:

Nếu PUB: <Badge className="bg-green-100 text-green-700">Công khai</Badge>

Nếu CONF: <Badge className="bg-yellow-100 text-yellow-700">Nội bộ</Badge>

Nếu SEC: <Badge className="bg-red-100 text-red-700 border-red-500 shadow-md"><LockIcon/> Tuyệt mật</Badge>. Đồng thời set class viền thẻ dự án nhãn SEC thành màu đỏ nhạt để tăng tính trực quan.

4.6. Bảng Điều khiển Kiểm toán (Audit Dashboard - FGA)
Route: /audit

Quyền truy cập: Wrap toàn bộ trang bằng component bảo vệ, nếu role !== 'HR_MANAGER' thì redirect về trang 403.

UI Components: Bảng dữ liệu log chuyên sâu mang phong cách Admin (Terminal-like). Cột hiển thị: Thời gian, User_DB, Thao tác, Bảng vi phạm, và Cột "Lệnh SQL" (được wrap trong thẻ <code> có màu nền xám đen).

5. UI COMPONENTS CHUNG BẮT BUỘC (SHARED COMPONENTS)
AI cần khởi tạo các Higher-Order Components (HOC) và tiện ích sau:

RoleGuard: <RoleGuard allowedRoles={['HR_MANAGER', 'HR_STAFF']}> ... </RoleGuard>. Dùng để bọc các phần tử UI nhạy cảm (nút xóa, nút sửa). Nếu role hiện tại không khớp, trả về null.

SecurityToast: Tích hợp từ shadcn/ui Toast. Định dạng riêng cho lỗi 403/Bảo mật (Màu nền đỏ sẫm, icon cái khiên cảnh báo ShieldAlertIcon).
